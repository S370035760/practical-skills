# 多平台内容一键发布

## 核心功能

- **一键多平台发布** - 写一次内容，自动发布到多个平台
- **智能格式适配** - 自动调整标题长度、图片尺寸、标签格式
- **发布状态追踪** - 统一查看各平台发布状态和数据
- **定时发布** - 支持设置最佳发布时间

## 支持的平台

| 平台 | 标题限制 | 图片要求 | 标签格式 | 特色功能 |
|------|---------|---------|---------|---------|
| 微信公众号 | ≤64 字 | 封面 900x383px | #话题# | 草稿/预览 |
| 知乎 | ≤20 字 | 封面 1200x675px | #话题# | 专栏/回答 |
| 掘金 | ≤50 字 | 封面 1280x720px | #标签# | 分类选择 |
| 小红书 | ≤20 字 | 封面 3:4 竖图 | #标签# | 表情优化 |
| 微博 | ≤140 字 | 9 图限制 | #话题# | @提及 |

## 快速开始

### 基础用法

告诉我要发布的内容和目标平台：

```
帮我把这篇文章发布到公众号和知乎

标题：AI 自动写周报的 5 个技巧
内容：[文章内容...]
图片：[图片路径...]
```

### 指定平台选项

```
发布到小红书，要求：
- 标题加 emoji
- 图片自动裁剪为 3:4
- 添加热门标签
- 下午 6 点发布
```

## 平台格式规则

### 微信公众号

**标题：**
- 最长 64 字（超过自动截断）
- 避免敏感词（政治、色情、暴力）
- 可添加 emoji 增加点击率

**图片：**
- 封面图：900x383px（2.35:1）
- 正文图片：宽度自适应，建议≤1080px
- 单图大小≤5MB

**标签：**
- 支持 #话题# 格式
- 最多 3 个标签

**发布模式：**
- `draft` - 保存到草稿箱（默认）
- `publish` - 直接发布（需确认）

### 知乎

**标题：**
- 最长 20 字（严格限制）
- 避免标题党
- 可添加疑问句增加互动

**图片：**
- 封面图：1200x675px（16:9）
- 正文图片：支持 GIF
- 单图大小≤10MB

**标签：**
- 支持 #话题# 格式
- 最多 5 个话题

**发布模式：**
- `article` - 发布文章
- `answer` - 回答问题（需指定问题链接）
- `column` - 发布到专栏

### 掘金

**标题：**
- 最长 50 字
- 技术类内容优先
- 可添加【分类】前缀

**图片：**
- 封面图：1280x720px（16:9）
- 支持 Markdown 图片语法
- 单图大小≤5MB

**标签：**
- 支持 #标签# 格式
- 必须选择技术分类（前端/后端/AI 等）

### 小红书

**标题：**
- 最长 20 字（严格）
- 必须添加 emoji（至少 2 个）
- 避免硬广词汇

**图片：**
- 封面图：3:4 竖图（1080x1440px）
- 最多 9 张图
- 首图最重要

**标签：**
- 支持 #标签# 格式
- 最多 10 个标签
- 热门标签增加曝光

**文案特点：**
- 多用 emoji 分隔段落
- 口语化表达
- 添加"干货""必看"等引导词

### 微博

**标题：**
- 无单独标题，正文前 140 字最重要
- 可添加话题 #话题# 作为标题

**图片：**
- 最多 9 张图
- 支持 GIF
- 单图大小≤5MB

**标签：**
- 支持 #话题# 格式
- 支持 @用户 提及

## 高级功能

### 智能标题优化

根据平台特点自动优化标题：

```
原标题：AI 自动写周报的方法

→ 公众号：🤖 AI 自动写周报的 5 个技巧，每天早下班 1 小时
→ 知乎：如何用 AI 自动写周报？
→ 小红书：🤖打工人必备！AI 写周报真香✨
→ 微博：#AI 写周报# 这个方法让我每天早下班 1 小时！
```

### 图片自动处理

- 自动裁剪为平台要求的尺寸
- 压缩至合适大小
- 添加水印（可选）
- 生成多尺寸版本

### 定时发布

设置最佳发布时间：

```
定时发布设置：
- 公众号：工作日 9:00（早高峰）
- 知乎：工作日 20:00（晚高峰）
- 小红书：周末 18:00（流量高峰）
- 微博：全天多时段（8:00/12:00/18:00/22:00）
```

### 发布数据追踪

发布后自动追踪：

- 阅读量/播放量
- 点赞/收藏/评论数
- 粉丝增长
- 各平台数据对比

## 脚本使用

### 发布脚本

```bash
# 基础发布
node scripts/publish-to-social.js \
  --platform wechat \
  --title "文章标题" \
  --content "文章内容.md" \
  --images ./images/ \
  --mode draft

# 多平台发布
node scripts/publish-to-social.js \
  --platform wechat,zhihu,juejin \
  --title "文章标题" \
  --content "文章内容.md" \
  --schedule "2026-03-10T09:00:00+08:00"
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--platform` | 目标平台（逗号分隔） | 必填 |
| `--title` | 文章标题 | 必填 |
| `--content` | 文章内容文件 | 必填 |
| `--images` | 图片文件夹路径 | 可选 |
| `--mode` | 发布模式（draft/publish） | draft |
| `--schedule` | 定时发布时间（ISO8601） | 立即 |
| `--tags` | 标签（逗号分隔） | 自动生成 |

## 注意事项

### 内容安全

- ❌ 避免政治敏感内容
- ❌ 避免色情暴力内容
- ❌ 避免虚假广告
- ✅ 遵守各平台社区规范

### 发布频率

- 公众号：建议每日≤1 篇
- 知乎：建议每日≤3 篇
- 掘金：建议每周≥2 篇
- 小红书：建议每日 1-2 篇
- 微博：无限制（但避免刷屏）

### 账号配置

首次使用需要配置各平台账号信息：

```bash
# 创建配置文件
cp .env.example .env

# 编辑 .env 文件，填入各平台 token
# WECHAT_TOKEN=xxx
# ZHIHU_TOKEN=xxx
# JUEJIN_TOKEN=xxx
# XIAOHONGSHU_TOKEN=xxx
# WEIBO_TOKEN=xxx
```

## 常见问题

**Q: 发布失败怎么办？**

A: 检查以下几点：
1. 账号 token 是否有效
2. 内容是否违反平台规范
3. 图片尺寸是否符合要求
4. 网络连接是否正常

**Q: 如何查看发布状态？**

A: 使用以下命令：
```bash
node scripts/publish-status.js --date 2026-03-09
```

**Q: 可以批量发布历史文章吗？**

A: 可以，使用批量模式：
```bash
node scripts/batch-publish.js --folder ./articles/ --platform all
```
