# 各平台详细配置参考

## 微信公众号

### API 接入方式

**官方 API：**
- 文档：https://developers.weixin.qq.com/doc/offiaccount/Getting_Started/Overview.html
- 需要：公众号 AppID + AppSecret
- 权限：需要认证的服务号

**第三方工具：**
- 微小宝、新榜、壹伴等
- 优点：无需开发，直接可用
- 缺点：功能受限，有费用

### 内容规范

**标题：**
- 最长 64 字
- 避免：政治敏感、色情暴力、虚假广告
- 推荐：数字 + 利益点 + 情绪词

**封面图：**
- 头条：900x383px（2.35:1）
- 次条：200x200px（1:1）
- 格式：JPG/PNG
- 大小：≤5MB

**正文：**
- 支持 Markdown 转换
- 图片宽度建议≤1080px
- 视频支持 MP4 格式

**发布时间：**
- 最佳：工作日 8:00-9:00、12:00-13:00、20:00-21:00
- 避免：深夜 23:00-6:00

## 知乎

### API 接入方式

**官方 API：**
- 文档：https://www.zhihu.com/developers
- 需要：申请开发者权限
- 限制：审核严格，个人较难申请

**替代方案：**
- 浏览器自动化（Selenium/Puppeteer）
- 模拟登录 + Cookie
- 风险：可能被封号

### 内容规范

**标题：**
- 最长 20 字（严格）
- 推荐：疑问句、对比句、数字
- 避免：标题党、夸大其词

**封面图：**
- 推荐：1200x675px（16:9）
- 最小：600x338px
- 格式：JPG/PNG/GIF
- 大小：≤10MB

**内容：**
- 支持 Markdown
- 支持代码块高亮
- 支持视频嵌入

**发布时间：**
- 最佳：工作日 20:00-22:00
- 周末：14:00-16:00

## 掘金

### API 接入方式

**官方 API：**
- 文档：https://juejin.cn/help/api
- 需要：登录 Cookie
- 权限：注册开发者即可

**API 端点：**
```
POST https://api.juejin.cn/content_api/v1/content/article/create
```

### 内容规范

**标题：**
- 最长 50 字
- 推荐：【分类】+ 技术点 + 收益
- 示例：【前端】React 性能优化的 10 个技巧

**封面图：**
- 推荐：1280x720px（16:9）
- 格式：JPG/PNG
- 大小：≤5MB

**分类：**
- 前端、后端、Android、iOS、AI
- 工具、代码人生、阅读

**标签：**
- 最多 5 个
- 必须选择技术分类

**发布时间：**
- 最佳：工作日 9:00-10:00、14:00-15:00
- 技术文章：周二至周四效果最好

## 小红书

### API 接入方式

**官方 API：**
- 不对外开放
- 需要：企业号 + 特殊申请

**替代方案：**
- 浏览器自动化
- 手机自动化（Appium）
- 第三方工具（灰产风险）

### 内容规范

**标题：**
- 最长 20 字（严格）
- 必须：添加 emoji（至少 2 个）
- 推荐：痛点 + 解决方案 + 情绪词

**封面图：**
- 推荐：1080x1440px（3:4 竖图）
- 最小：750x1000px
- 格式：JPG/PNG
- 大小：≤5MB
- 数量：1-9 张

**文案：**
- 口语化表达
- 多用 emoji 分隔段落
- 添加互动引导（"收藏""点赞"）

**标签：**
- 最多 10 个
- 推荐：2-3 个大流量标签 + 3-5 个精准标签

**发布时间：**
- 最佳：工作日 18:00-20:00
- 周末：10:00-12:00、19:00-21:00

## 微博

### API 接入方式

**官方 API：**
- 文档：https://open.weibo.com/wiki/微博 API
- 需要：微博开放平台应用
- 权限：个人可申请

**API 端点：**
```
POST https://api.weibo.com/2/statuses/update.json
```

### 内容规范

**正文：**
- 最长 140 字（超过折叠）
- 重要信息放前面
- 可添加话题 #话题#

**图片：**
- 最多 9 张
- 支持 GIF
- 大小：≤5MB/张

**视频：**
- 最长 15 分钟
- 支持横屏/竖屏

**发布时间：**
- 最佳：8:00-9:00、12:00-13:00、18:00-19:00、22:00-23:00
- 高频：可多次发布（但避免刷屏）

## 通用最佳实践

### 内容适配策略

1. **一源多用** - 写一次核心内容，适配各平台
2. **平台特色** - 根据平台调性调整语气和格式
3. **发布时间** - 各平台流量高峰不同，分别定时
4. **数据追踪** - 统一查看各平台表现

### 避免封号

1. **不要频繁操作** - 每平台每日发布≤5 次
2. **不要完全相同** - 各平台内容要有差异化
3. **不要违规内容** - 遵守各平台社区规范
4. **使用官方 API** - 避免被识别为机器人

### 数据指标

**核心指标：**
- 阅读量/播放量
- 点赞数
- 收藏数
- 评论数
- 转发数
- 粉丝增长

**转化指标：**
- 点击链接数
- 私信咨询数
- 付费转化数
